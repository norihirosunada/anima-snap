# **アニミズム・スナップ (Animism Snap) 要件定義書**

「無機物に魂を宿し、日常を物語に変えるライフログ・エンターテインメント」

## **1\. ユーザーフロー (Object Interaction)**

ユーザーが「モノ」をカメラで捉え、対話や記録を行う際のフローです。

### **1.1 初期スキャンと覚醒 (Initial Awakening)**

初めてのモノをコレクションに加える際のフロー。

1. **スキャン (Live View):** ユーザーが対象物をカメラで動画で映し続ける。  
2. **判定 (Recognition):** 映像からキャプチャした特定の瞬間の画像を **Gemini 3.1 Pro** が解析。  
   * **同一判定チェック:** 既存のコレクションデータと照合し、既に登録済みの個体（傷、ステッカー、形状の微細な一致）かどうかを判定。  
   * 新規個体と判定された場合、以下の覚醒プロセスへ.  
3. **並行実行プロセス (Parallel Processing):**  
   * **A. 感情抽出 & 親愛度注入 (Affection Sync):**  
     * **ビジュアル・アフェクション認識:** 映像から「撫でる」「触れる」といったアクションをリアルタイムに親愛度（Affinity）へ変換（Multi-modal Live API等を使用）。  
     * **低負荷な質問回答:** 愛情表現の合間に、モノへの思い入れを聞く短い質問にクイックリプライで回答。  
   * **B. 覚醒動画生成:** **Veo 3.1** を用い、静止物が動き出す演出動画を生成。  
4. **覚醒演出 (The Awakening):** カメラ映像に生成された演出動画をオーバーレイ再生し、魂が宿る瞬間を可視化。  
5. **キャラクター定義 (Persona):** Gemini 3.1 Pro が熱量と回答から性格・口調・記憶を生成。  
6. **コレクション登録:** ローカルストレージに保存し、最初の挨拶が行われる。

### **1.2 再訪と追加の思い出 (Re-encounter & Memory Recording)**

既に登録済みのモノを再度スキャンした際のフロー。

1. **再認識:** **Gemini 3.1 Pro** が「これは登録済みの〇〇（例：NoriのMacBook）」であると特定。  
2. **再会演出 (Re-awakening):** 初回同様、**Veo 3.1** を用いて動き出し動画を生成。キャラクターが再び魔法のように動き出す演出を行う。  
3. **思い出の記録 (Memory Capture):** \* スキャン中の現在の状況（場所、時間、周囲の音、ユーザーの表情）をAIが解析。  
   * 「今日はカフェで作業しているんですね」「新しいステッカーが増えましたか？」など、変化や状況に基づいた会話が発生。  
4. **同期率（親愛度）の更新:** スキャン中のコミュニケーション（撫でる等）や、新しいエピソードの共有により、親密度が上昇。  
5. **ライフログ統合:** その瞬間のスナップショットや対話内容を、そのモノの「追加の思い出」としてローカルストレージのタイムラインに蓄積。

## **2\. 画面要件 (UI/Screen Requirements)**

### **2.1 キャプチャ画面 (Capture Screen)**

「発見」と「覚醒」の儀式を行う画面。

* **カメラビュー:** フルスクリーンの背面カメラ映像。常に動画として映し続け、バックグラウンドで抽出した静止画を用いてリアルタイムにオブジェクトを認識。  
* **親愛度認識エフェクト:** 映像内でユーザーの手がモノに触れたり、撫でたりする動作を認識した箇所から、オーラが溢れ出す演出。  
* **感情抽出モーダル:** 回答を集めるための対話型UI。クイックリプライボタンをタップするだけでスムーズに回答できるインターフェース。  
* **Awakening Player:** Veo 3.1 で生成されたシネマティックな動画をオーバーレイ再生するレイヤー。

### **2.2 モノコレクション一覧 (Collection List)**

宿った魂たちの安らぎの場。

* **カード/グリッド表示:** 登録済みのモノたちのポートレート。

### **2.3 チャット画面 (Chat Screen)**

「程よい距離感」を実現する対話インターフェース。

* **ログ履歴:** 過去の対話や、モノが勝手に記録した「ライフログ（観測記録）」のタイムライン。再訪時の思い出もここに蓄積される。

### **2.4 モノ詳細画面 (Object Detail)**

対象への理解を深める環境設定やパーソナル・プロファイル。

* **3D/2Dビジュアル:** モノの「真の姿（精霊化後）」のグラフィック。  
* **パーソナリティ概要:** Gemini 3.1 Pro が生成した深い洞察に基づく性格説明、二つ名。  
* **データ統計:** 使用時間、観測回数、一緒に過ごした日数、蓄積された思い出の数。

## **3\. 技術スタック・アーキテクチャ**

| コンポーネント | 技術選定 | 役割 |
| :---- | :---- | :---- |
| **Frontend** | React / Tailwind CSS | UI/UX, カメラ制御, 動画オーバーレイ |
| **Graphics** | p5.js / Three.js | 精霊のエフェクト, 愛情表現への視覚的フィードバック |
| **VLM/LLM** | **Gemini 3.1 Pro / Live API** | 画像解析, **同一個体の識別**, 性格定義, 対話生成 |
| **Generation** | **Veo 3.1** | オブジェクトが動き出すシネマティックな動画生成 |
| **Storage** | **Local Storage** | コレクションデータ, ユーザープロフィール, 共有記憶の永続化 |

## **4\. データ構造 (Storage Schema)**

ブラウザの localStorage を使用し、JSON形式でデータを保持します。

* **Collections (Key: animism\_snap\_collections):** \* { objectId: { name, type, personality, affinity, awakeningVideoUrl, ... } }  
* **Memories (Key: animism\_snap\_memories):** \* \[ { objectId, timestamp, type: "awakening" | "re-encounter", content, snapshotUrl }, ... \]

## **5\. ハッカソン向けMVP優先順位**

1. **Gemini 3.1 Pro によるリアルタイム解析と対話:** 動画内フレーム解析によるキャラクター生成を優先。  
2. **映像ベースの親愛度認識インタラクション:** 手の動きを認識したパーティクル演出とクイックリプライの統合。  
3. **Veo 3.1 による覚醒演出のプロトタイプ:** 最も魔法のような瞬間の実現。
